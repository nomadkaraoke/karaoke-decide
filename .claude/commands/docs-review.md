# Documentation Review Command

Review and update documentation after changes.

## Instructions

1. Check which files were recently modified
2. Review relevant documentation:
   - docs/README.md - status and quick links
   - docs/ARCHITECTURE.md - if architecture changed
   - docs/API.md - if API endpoints changed
   - docs/DEVELOPMENT.md - if dev workflow changed
   - PLAN.md - if requirements changed

3. Update any outdated information
4. Mark completed items in docs/README.md

## Checklist

- [ ] README status is current
- [ ] API docs match implementation
- [ ] Architecture diagrams are accurate
- [ ] Development guide is up to date
